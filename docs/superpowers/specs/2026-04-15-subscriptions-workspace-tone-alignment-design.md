# Subscriptions Workspace Tone Alignment Design

## Goal

購読一覧ページと購読の整理ページを、記事閲覧ページの暗く静かなトーンと、設定ページの整列感に揃える。特に左ペインは RSS リーダーらしい軽い一覧性を取り戻し、右ペインは詳細を読むための静かな面として維持する。

## Current Problem

- 購読一覧の左ペインがカード化されすぎていて、元のフィードツリーより重い
- 記事閲覧ページと設定ページに比べて、購読一覧/整理だけ境界線、面、余白の扱いが別系統に見える
- 管理画面としての情報設計は進んだが、RSS クライアントらしい軽いスキャン性が弱い
- 購読整理は左リストと右詳細の役割が分かれてきた一方で、左側の一覧体験はまだ重い

## Design Direction

### Visual Base

- ベースは記事閲覧ページの暗さ、静かな余白、細い罫線
- 情報行と整列は設定ページのルールを流用する
- 画面全体を「管理専用ワークスペース」として扱い、通常の左サイドバーは表示しない
- 左は一覧、右は詳細という責務を明確に分ける

### Tone Mapping

- 左ペイン:
  軽量、流し見しやすい、構造が分かる
- 右ペイン:
  静かなカード面、情報の整列、判断理由の強調
- 共通部品:
  角丸、border、surface tone、chip/button の高さを揃える

## Left Pane Design

### Row Model

左ペインは 3 種類の行で構成する。

1. `Folder Row`

- 軽い行
- chevron、フォルダ名、右に件数
- hover はごく薄い背景
- ドロップターゲット時のみ受け入れ表現を出す

1. `Feed Row`

- 通常時はフラット
- 左に avatar または favicon
- タイトル + 補助情報 1 行
- 面は持たせすぎず、軽く並ぶことを優先

1. `Selected Feed Row`

- 基本構造は通常行と同じ
- 背景トーン差と薄い border で一段だけ強調
- 「カード」ではなく「選択されたツリー行」に留める

### Tree Behavior

- 既存のフィードツリー表現と drag/drop ロジックを優先して再利用する
- 最初の対象は `フォルダなし -> フォルダ`
- UI としては、将来的にすべてのフィードが移動可能に見えても違和感がない構造にする
- drag hover は背景ベタ塗りより、薄い outline や subtle highlight を優先する

### Scrolling

- 左ペインは独立スクロール
- フォルダ/フィード一覧が長くなっても、下まで到達できることを最優先とする
- 右ペインとスクロール責務を分離する

## Right Pane Design

### Shared Detail Surface

- 購読一覧と購読整理の右ペインは同じ detail card family を使う
- 詳細面は情報カードとして静かに見せ、左の一覧より情報密度を高くする
- URL 行は独立したラベルとして並べず、タイトルをリンクにする

### Message Treatment

- `最近も動きがあり、このまま残してよさそうです。` のような文は本文ではなく、システムメッセージボックスとして扱う
- 罫線は薄く、背景はわずかなトーン差のみ
- badge/chip より一段大きいが、alert ほど重くない見た目にする

### Sections

- メトリクス行は設定ページ寄りの整列感で見せる
- `最近の記事` セクションは、メトリクス群から一段下げて独立させる
- 見出し直下は詰め、上との間だけ広く取って hierarchy を見せる
- 記事行はタイトルと日付の縦位置を整える

### Action Button

- `購読の整理で開く` は細いアウトライン CTA として共通化する
- 丸すぎない角丸を採用する
- 行内アクションや整理キューのアクションと family resemblance を持たせる

## Cleanup Page Specifics

### Header / Summary

- 上部は `確認待ち / 判断済み + フィルタチップ列` をベースとする
- 情報カードを横に並べるのではなく、密度の高い control strip として扱う

### Decision Actions

- 判断操作は右詳細依存ではなく、左リストの流れの中で完結できるようにする
- 上部バーと行内ボタンに `残す / あとで / 削除` を置く
- 右詳細は「読む」責務に寄せる

## Subscriptions Index Specifics

### Summary Cards

- 件数が `0` のカードはグレーで非活性
- 件数が閾値以上あるカードだけ色を付けてクリック可能にする
- クリック領域はカード全体

### Relationship to Original Feed Tree

- v0 が参照していた元のフィード/カテゴリ UI に近づける
- 特に一覧の軽さ、フォルダ階層、件数表示は元のツリー体験を尊重する
- 「通常は軽いツリー、選択時だけ少し強調」という中間案を採用する

## Data Flow

- 左ペイン選択 → 右詳細同期
- drag/drop 成功 → 左一覧の所属がその場で更新
- 右詳細は選択中フィードの状態を表示
- 購読整理へ遷移して戻る場合、選択対象と文脈をなるべく保持する

## Risks

1. 既存ツリーの drag/drop を流用する際に、購読一覧側の表示責務と混ざる
対応:
一覧専用 adapter 層で吸収し、ドラッグ処理本体は再実装しない

2. 選択強調を強くしすぎると、またカード一覧に逆戻りする
対応:
強調は background tone + thin border に限定する

3. 右詳細がリッチすぎると左とのトーン差が広がる
対応:
右側は余白、線、タイポを整理して静かに保つ

## Validation

- 左ペインが独立スクロールする
- フォルダ/フィードの階層が軽く読める
- 選択中だけ一段強調される
- drag/drop でフォルダ移動できる
- 右詳細が購読一覧/整理で同じ family に見える
- `mise run check` が通る
