# Reader Pane Surface Simplification Design

## Goal

記事を読むペインからカード感と枠線の主張を減らし、Reeder に近い静かな読み面へ寄せる。
ただし色まで合わせることは目的にせず、主に以下を調整対象とする。

- 情報の強弱
- タイトルとメタ情報のサイズ感
- 余白による境界づけ
- 画像と本文の並び順

## Non-Goals

- 記事選択、表示モード切り替え、リンク遷移などの挙動変更
- 記事ペイン以外の sidebar / article list / browser overlay の再設計
- データ取得や state 管理の変更
- Reeder の文字色、配色、タイポグラフィの完全再現

## Current Problem

- 上部メタ情報が大きなカード面に入っており、読み始める前に UI の存在感が勝っている
- feed 名がピル表現になっていて、補助情報よりコントロールに見える
- タグがタイトル直後の区切り領域に置かれており、読みの導線を分断している
- タイトル hover の面積が大きく、静かな読み面よりインタラクティブなカードに寄っている
- メタ情報ブロックと本文ブロックが「上下に積まれた別パネル」に見え、記事全体が一続きの面として読めない

## Approved Direction

採用案は `Editorial Minimal` をベースに、必要最低限の整理感だけ `Soft Sections` から借りる構成とする。

- 構成は `日付 → タイトル → 出典/著者 → 画像 → 本文`
- 上部カードは廃止し、背景面の上に情報をそのまま置く
- 区切りは border ではなく余白を基本とする
- メタ情報の終端に限って、ごく薄い 1 本線を入れる
- 本文幅は現状より少しだけ絞り、カード削除後の間延びを防ぐ
- タグは主役から外し、本文直前または本文末の補助情報として扱う

## Visual Rules

### Hierarchy

- 主役はタイトル
- その次に公開日時
- feed 名と著者は補助情報として同一レベルで弱く表示する
- タグはさらに一段弱い情報として扱う

### Spacing

- 情報の切り分けは `面` ではなく `縦余白` で行う
- タイトルの上は詰めすぎず、日時との間に呼吸を置く
- タイトルと補助情報の間はやや狭く、タイトルと画像の間は広めに取る
- 画像と本文の間も、別カードに見えない範囲で十分な余白を取る

### Surface Treatment

- `rounded-[2rem] border shadow gradient` 系のカード演出は外す
- feed 名の pill / chip 表現はやめ、テキストまたはごく軽いボタン表現へ落とす
- タイトルクリックの hover は「面で囲う」より「わずかなトーン変化」に留める
- 本文画像の角丸は残し、UI カードの角丸とは切り離して扱う

## Layout Model

### Article Header

記事ヘッダーは独立カードではなく、本文の導入部として扱う。

- 1 行目: 公開日時
- 2 行目: タイトル
- 3 行目: feed 名と著者
- 4 行目以降: 必要に応じてタグ

この順序により、最初の視線は常にタイトルへ落ち、その後に出典情報を拾える。

### Article Image

- 画像はタイトルの後ろへ置く
- 画像がない場合もヘッダーと本文の距離感が崩れないようにする
- 画像はコンテンツの一部として置き、独立パネルにはしない

### Article Body

- 本文は header と同一カラムに揃える
- 本文幅は現状より少し絞る
- 読みやすさ優先で、行長が広がりすぎないようにする
- prose スタイルは維持しつつ、header との調和を優先する

## Component Scope

主な変更対象は以下の 3 コンポーネントに限定する。

- `src/components/reader/article-reader-body.tsx`
- `src/components/reader/article-meta-view.tsx`
- `src/components/reader/article-content-view.tsx`

### `article-reader-body.tsx`

- 上部の囲み `section` を廃止する
- メタ情報、タグ、画像、本文を単純な縦積みに再構成する
- 全体カラム幅と上下余白を調整し、カードなしでも間延びしない読み面にする

### `article-meta-view.tsx`

- feed 名の pill 表現を弱める
- タイトル hover 領域を縮小またはトーンダウンする
- 日付、タイトル、補助情報の視覚的な序列を明確にする
- 著者なしでもレイアウトが破綻しないようにする

### `article-content-view.tsx`

- 画像の前後余白を見直す
- 画像角丸は維持し、header と分離した単独コンテンツとして扱う
- 本文の見え方は大きく変えず、上部の静かな構成に繋がるよう微調整する

## Interaction and Behavior

- タイトルクリックによるリンク遷移は維持する
- middle click / modifier key の挙動も維持する
- feed 名クリックによる feed 選択も維持する
- タグ表示の場所は変えてよいが、タグ機能自体は変えない

つまり今回の変更は、見た目の責務だけで完結させる。

## Edge Cases

特に以下のケースで縦積み構造が崩れないことを優先する。

- 長いタイトル
- 著者なし
- 画像なし
- タグが多い
- 日本語と英語で文字面が異なるケース

各ケースでも、タイトルが主役であることと、本文への導線が分断されないことを重視する。

## Risks

1. フラットに寄せすぎると情報の区切りが消え、出典やタグが埋もれる
対応:
余白主体で整理しつつ、メタ情報終端の薄い 1 本線で区切りを固定する

2. タイトル hover を弱めすぎるとクリック可能性が落ちる
対応:
hover は残しつつ、カードのような大面積演出だけ消す

3. カード削除後に全体が広く見えすぎ、読み面が散る
対応:
本文幅を少し絞り、タイトルから本文までを同一の静かなカラムへ収める

## Validation

- 記事ペイン上部の大きなカードが消えている
- 見出しの視線誘導が `日付 → タイトル → 出典/著者` の順に自然である
- feed 名が chip ではなく補助情報として見える
- タグが主役を邪魔しない位置へ移っている
- 長いタイトル、著者なし、画像なしでも縦積みが崩れない
- 既存の article 系テストは維持し、必要な差分のみ更新する
- 最終的に `mise run check` を通す
