# UI デザインレビュー反復ルール

## 対象

- UI 実装、UI 修正、見た目に影響するフロントエンド変更全般

## 完了前の必須フロー

- 実装ができたら、そのまま完了扱いにせずデザインレビューを行う
- レビューでは `DESIGN.md` との整合、情報階層、余白、視線誘導、状態の分かりやすさ、操作導線、文言、レスポンシブ崩れを確認する
- 100 点満点の自己採点で 95 点以上に達していない場合は、改善点を洗い出して修正する
- 修正後に再レビューし、95 点以上になるまでこのループを繰り返す

## 実務上の基準

- 95 点未満のまま「完了」と報告しない
- 見た目の評価はコードだけで済ませず、可能な限り実画面で確認する
- UI 変更の完了報告では、どの観点をレビューして何を直したかを短く添える
- 制約や既存仕様の都合で 95 点相当まで上げきれない場合は、妥協点と残課題を明示する
