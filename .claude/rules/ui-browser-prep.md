# UI ブラウザ確認ルール

## UI/UX 調整の開始手順

- UI の余白、配置、視覚状態を調整する時は、コード変更前に必ずブラウザで実画面を開いて確認する
- 開発サーバーは `pnpm dev --host 127.0.0.1 --port 4173 --strictPort` で起動する
- 確認 URL は `http://127.0.0.1:4173/` を使う
- `src/main.tsx` の `setupDevMocks()` により、Tauri 外ではブラウザ用の mock IPC が自動注入される。UI 確認はまずこのブラウザモードで行う

## dev-browser の使い方

- UI 確認と調整には `dev-browser` を優先して使う
- `dev-browser` の依存が未導入なら先に `dev-browser install` を実行する
- ページは `browser.getPage("ultra-rss-reader")` のような名前付きページで開き、同じページを再利用して状態を保持する
- 要素特定には `snapshotForAI()` を使い、必要に応じてスクリーンショットも併用する

## 調整時の必須フロー

- 変更前に対象画面を表示し、どの要素を触るのかを実画面で特定する
- 変更後は同じ画面を再表示して、見た目の差分を必ず確認する
- UI 変更の説明では、どの画面をどの方法で確認したかを明記する

## 避けること

- UI の見た目をコードだけで推測して修正を確定しない
- まず Tauri アプリを立ち上げる前提で進めない。最初の確認はブラウザモードを優先する
